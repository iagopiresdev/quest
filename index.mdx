# Quest

`quest` is a Bun-native, CLI-first orchestration system for spec-driven agent work.

It is designed around a few non-negotiable constraints:
- specs drive planning and execution
- workers are typed and calibratable
- runs stay local and inspectable
- integration is conservative
- observability is event-driven, not bolted on afterward

## Current Capabilities

- plan quests from typed specs
- register workers with backend metadata, trust, progression, and calibration history
- execute runs through `local-command`, `codex-cli`, and `hermes-api`
- integrate completed runs through dedicated workspaces
- emit typed observable events and dispatch them to sinks such as webhook and Telegram

## Local Docs Preview

Mintlify expects a `docs.json` file at the project root. To run the docs locally:

```bash
npx mint dev --no-open
```

Or use the repo script:

